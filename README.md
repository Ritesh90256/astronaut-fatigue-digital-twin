# 🚀 Astronaut Fatigue & Sleep Deprivation Digital Twin

A digital twin system that simulates and monitors the physiological and cognitive effects of fatigue and sleep deprivation during a 30-day space mission.

Built by **Ritesh** as part of a simulation and modelling project.

---

## 📋 Project Overview

This project models an astronaut's physical and cognitive state over a 30-day mission using:
- **Real human physiology** from the BioGears engine (US Army validated)
- **Probabilistic mission simulation** with discrete events (EVA, sleep interruptions, workload spikes)
- **Monte Carlo analysis** across 1000 mission runs to quantify fatigue risk
- **Unity 3D digital twin** that visually represents the astronaut's health state in real time

## ✨ Features

- ✅ Real BioGears physiological baseline (HR, SpO2, BP, Respiration Rate)
- ✅ 30-day mission simulation with 4 mission phases
- ✅ Probabilistic events (EVA spacewalks, sleep interruptions, motion sickness)
- ✅ Monte Carlo analysis (1000 runs, nominal vs high-stress scenarios)
- ✅ BiologicalDebt modelling (Van Dongen sleep homeostasis model)
- ✅ AttentionLapses modelling (Dinges PVT model)
- ✅ Unity 3D astronaut that changes colour green → yellow → red
- ✅ Real-time vital signs display (Heart Rate, SpO2, Respiration Rate)
- ✅ Streamlit interactive dashboard
- ✅ SSH bridge between Windows and Linux for BioGears integration

## 🚀 How to Run

### Prerequisites
- Windows 10/11
- VMware Workstation with Ubuntu 22.04 VM
- Python 3.11
- Unity 6
- BioGears built from source on Ubuntu VM

### Step 1 — Start BioGears (Ubuntu VM)
```bash
cd ~/biogears/build/runtime
export LD_LIBRARY_PATH=~/biogears/install/bin:~/biogears/install/bin/howtos
~/biogears/install/bin/bg-scenario --help
```

### Step 2 — Fix VMware Network (Windows PowerShell as Admin)
```powershell
net stop vmnetdhcp && net stop vmnat
net start vmnetdhcp && net start vmnat
netsh interface set interface "VMware Network Adapter VMnet8" disable
netsh interface set interface "VMware Network Adapter VMnet8" enable
```

### Step 3 — Test SSH Connection
```bash
python -c "import paramiko; c=paramiko.SSHClient(); c.set_missing_host_key_policy(paramiko.AutoAddPolicy()); c.connect('192.168.47.128', username='YOUR_USERNAME', password='YOUR_PASSWORD'); print('SSH OK'); c.close()"
```

### Step 4 — Run Python Simulation
```bash
pip install paramiko matplotlib streamlit numpy
python main.py
```

### Step 5 — Run Streamlit Dashboard
```bash
streamlit run visualization/dashboard.py
```

### Step 6 — Run Unity Digital Twin
1. Copy `simulation_results.csv` to `AstronautFatigueDigitalTwin/Assets/StreamingAssets/`
2. Open Unity project
3. Press **Play ▶️**

## 🔬 BioGears Integration

BioGears runs on Ubuntu Linux inside VMware. Python on Windows connects via SSH (Paramiko) to:
1. Upload a scenario XML file to the VM
2. Run BioGears to simulate a StandardMale patient
3. Read the output CSV with real physiological values
4. Scale those values per day using validated sleep deprivation models:
   - **Van Dongen model** for biological debt accumulation
   - **Dinges PVT model** for attention lapses

## 📈 Mission Phases

| Phase | Days | Description |
|---|---|---|
| Launch Phase | 1–5 | High stress, body adapting to microgravity |
| Adaptation Phase | 6–15 | Moderate workload, sleep stabilising |
| Normal Operations | 16–25 | Routine tasks, occasional EVA events |
| High Workload Phase | 26–30 | Intensive operations, maximum fatigue |

## ⚠️ Important Notes

- Update `VM_HOST`, `VM_USER`, and `VM_PASSWORD` in `biogears/biogears_interface.py` for your VM
- BioGears must be built from source on Ubuntu 22.04 — see [BioGears GitHub](https://github.com/BioGearsEngine/core)
- The simulation falls back to analytical mode if the VM is unreachable
- `simulation_results.csv` is excluded from git (regenerated on each run)
